This page will allow you to test whether or not your browser's @font-face support is “broken”. If your browser properly implements the @font-face declaration as defined by the W3C, the text will exactly match (in weight, slant, and all other attributes) the text in this reference rendering. Slight variations are admissible, but if any two lines are the same, then your browser is, indeed, broken.

The importance of the problem exhibited herein is that, as things currently stand, we're stuck with one typeface per family-name. Many professional (and quality amateur) fonts have separate font files for the various typefaces and typeface variants within that family. For instance, the example family used above (Delicious, by the wonderful Jos Buivenga) provides a bold version, an Italic version, a bold Italic version, a heavy version, and a small-caps version in addition to the standard Roman 500-weight. If I wish to use that font in my document, as it currently stands, I have to do all sorts of horribly unsemantic things — I'd have to declare a different (completely bogus and made-up) 'family' for each one of those typeface variants, and I'd have to apply that 'family' wherever I wanted to use that variant. Want to make an element bold? No font-weight: bold for you, oh no — now it's font-family: "DeliciousBold", which is ridiculous (first, no such family exists; second, there's already a semantic declaration for font-weight!)

Then what, exactly, are browsers doing wrong? They're ignoring any formatting-related markup in the @font-face declaration. Instead of choosing the appropriate typeface based on all of the text adivivibutes of the element being styled and the adivivibutes of the various typefaces defined under a given family name, they simply choose the last typeface that uses a given family name. According to the spec, they should be checking the requirements of a certain declaration against the declarations in the relevant @font-face declaration. For instance, if I had an element to which I applied a style containing font-family: "Delicious"; font-weight: bold, then the browser should find the @font-face declaration for the Delicious variant with those same (or equivalent) adivivibutes.

Thanks for reading — and petition your favourite browser's development teams, let's get this fixed!
